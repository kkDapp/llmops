3 The 5 RAG Strategies

Each strategy is a different retrieval pipeline. The Model Router automatically selects Flash vs Pro vs fine-tuned LoRA based on query complexity patterns. All 5 are selectable from the UI or via the REST API.

StrategyHow it retrievesBest forLatency (cold)Cost
🔵 Naive Single vector similarity search → top-5 chunks → Gemini Simple factual questions ~7s Low
🟡 Advanced Query rewriting → multi-query retrieval → re-ranking → Gemini Ambiguous or complex queries ~12s Medium
🟠 Hybrid Dense vector search + sparse BM25 → score fusion → Gemini Queries with specific terms/entity names ~7s Low
🟣 Graph spaCy NER → entity graph (NetworkX) → graph traversal → Gemini Multi-hop reasoning, relationship queries ~7s Medium
🔴 Agentic Iterative retrieve-decide loop — Gemini decides if more retrieval needed Deep research, multi-step questions ~11-15s High

4 Kubernetes Production Architecture PRODUCTION

GKE cluster with Workload Identity (no service account keys), Horizontal Pod Autoscaler (2–10 pods, CPU 70%), KEDA scale-to-zero on Redis queue depth, and ingress via GKE Ingress with TLS termination. All secrets come from Kubernetes Secrets mounted as env vars — never baked into images.

📦 Deployment Strategy

  • Rolling update — zero-downtime deploys
  • maxSurge: 1, maxUnavailable: 0
  • readinessProbe on /health prevents bad rollouts
  • Resource requests: 500m CPU / 512Mi RAM
  • Resource limits: 2000m CPU / 2Gi RAM

📈 Autoscaling — Two Layers

  • HPA — Kubernetes native, scales 2→10 pods on CPU/Mem
  • KEDA — event-driven, scales to zero when Redis queue is empty
  • KEDA prevents paying for idle ingestion workers
  • Scale-up triggered in ~30s when Pub/Sub messages arrive

🔐 Workload Identity

  • Pod SA → GCP SA binding — no JSON keys anywhere
  • Scoped permissions: Vertex AI User + BigQuery Data Editor
  • Token auto-refreshed by GKE metadata server
  • ADC picks up identity transparently in Python SDK

📈 Evidently Drift Detection

  • Compares last 7-day query distribution vs baseline
  • Tests: Jensen-Shannon divergence on query embeddings
  • Detects topic shift — e.g., suddenly more legal queries
  • Outputs HTML report + JSON summary
  • JSON summary stored in MLflow artifacts
  • Triggers fine-tune if drift p-value < 0.05

🛡️ Prompt Injection Blocking

  • Detects "ignore previous instructions" patterns
  • Catches "forget your system prompt" variants
  • Blocks role-switching: "you are now DAN"
  • Case-insensitive regex matching
  • Returns HTTP 400 with sanitized error
  • Blocks counted in rag_guardrail_blocks_total

🔍 PII Redaction (Output)

  • Email: [\w.-]+@[\w.-]+\.\w+
  • Phone: US + international patterns
  • SSN: \d{3}-\d{2}-\d{4}
  • Credit Card: Luhn-like patterns
  • All replaced with [REDACTED] before API response
  • Runs on LLM output, not stored documents

🔐 Secrets — Zero Trust

  • No secrets in Docker images or Dockerfiles
  • .env is gitignored — never committed
  • Production: K8s Secrets → env vars at pod startup
  • GCP: Workload Identity — pods get token from metadata server
  • Gemini API key: env_file only, not in compose YAML

8 Async Ingestion Pipeline

Documents arrive from GCS, are chunked and embedded, and stored in ChromaDB. On-demand via the REST API for interactive use; at scale, GCS event triggers a Pub/Sub message which KEDA routes to an ingestion worker pool that scales to zero when idle.

Chunking strategy: Recursive character splitter at 512 tokens with 50-token overlap. Overlap ensures context isn't lost at chunk boundaries — a sentence that starts at the end of one chunk also appears at the start of the next. Metadata stored per chunk: source_uri, page_number, chunk_index, ingested_at.

9 Multi-Model Routing

Every query is routed to the right model tier by a rules engine that inspects query length, detected patterns, and load. Simple queries go to Flash (cheap, fast). Analytically complex queries go to Pro. After fine-tuning, domain-specific queries go to the LoRA adapter via vLLM. Cost multipliers are tracked per tier.

  • Query length > 200 chars → likely complex → Pro

10 Inference Optimization Layer

Four layers of optimization reduce latency and cost: semantic caching in Redis (1ms for cache hits), LiteLLM Redis prompt cache for identical prompts, vLLM continuous batching for self-hosted models, and QLoRA 4-bit quantization reducing the fine-tuned model size by 4×.
